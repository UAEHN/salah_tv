Privacy Policy for Ghasaq (غسق)

Last updated: April 4, 2026

Introduction

Ghasaq (غسق) is an Islamic prayer times app for Android TV and mobile devices. It displays prayer times, plays Adhan audio, streams Quran, and provides Adhkar and Qibla compass features. This policy explains how the app handles your data.

Most user data is stored locally on your device only. The app collects anonymous usage analytics to improve the user experience, but this data cannot be used to identify you personally.

Data We Do NOT Collect

The app does not collect, transmit, or share:

  • Names, email addresses, or account information
  • Device identifiers or advertising IDs
  • Any data with third parties for marketing or profiling

Anonymous Usage Analytics

The app uses Firebase Analytics (provided by Google) to collect anonymous usage data for the purpose of improving the app. The data collected includes:

  • Screens opened within the app
  • App features used (e.g., Adhan playback, Quran streaming, Tasbih, Qibla)
  • Settings changes (e.g., city selection, theme, layout) — without identifying values where possible
  • Session duration and engagement metrics
  • Device type (TV or mobile) as a non-personal user property

This data is anonymous and aggregated. It cannot be used to identify you personally. It is processed by Google in accordance with the Google Privacy Policy.

Data retention: Analytics and diagnostic data is automatically deleted by Google after a maximum of 14 months. Event-level data is deleted after 60 days by default.

You can opt out of analytics collection at any time through your device settings: Settings → Google → Ads → Opt out of Ads Personalization, or by resetting your advertising ID.

Location Data

The app requests location permission for two optional features:

  • Auto-detect city (mobile only): Uses GPS to find your nearest city and load the correct prayer times. The detected coordinates are saved locally on your device. This is optional — you can manually select your city instead.
  • Qibla compass (mobile only): Uses your device location to calculate the direction of Mecca. Location is only accessed while the Qibla screen is open and is never stored.

Location data is never sent to any server. It stays on your device.

Sensors

The Qibla compass uses the device magnetometer and accelerometer in real time to show compass direction. This sensor data is processed on-device only and is never logged or transmitted.

Local Storage

The app stores the following data locally on your device:

  • Your selected city, country, and optionally GPS coordinates
  • Prayer time settings (offsets, Iqama delays, calculation method)
  • App appearance preferences (theme, font, layout)
  • Adhan sound selection and Quran reciter preference
  • Notification settings per prayer
  • A cached list of Quran reciters (refreshed every 24 hours)

This data never leaves your device.

Internet Usage

The app requires internet access for the following purposes only:

  • Quran streaming: Audio is streamed on demand from mp3quran.net CDN servers. No personal data is sent.
  • Quran reciter list: A list of available reciters is fetched from https://mp3quran.net/api/v3/reciters and cached locally for 24 hours.
  • App update check: The app checks for a new version by fetching a small JSON file from GitHub. No personal data is sent.

Prayer Times Data

Prayer times for supported cities are bundled inside the app as a local database — no external prayer time API is used. For worldwide locations detected via GPS, times are calculated on-device using the adhan algorithm.

Notifications

On mobile devices, the app can send local notifications for prayer times, Adhan reminders, and Iqama times. These notifications are scheduled entirely on-device using Android's alarm system. No notification data passes through any external server.

User Feedback

The app includes an optional feedback feature (mobile only) that allows you to send a message to the developer. When you submit feedback, the following data is collected and stored:

  • Your feedback message (text you write)
  • Feedback type (e.g., bug report, suggestion)
  • Operating system platform (Android)
  • Submission timestamp

No personal information is collected — no name, email, account, or device identifier is attached to feedback submissions.

Feedback is stored in Cloud Firestore (Google Firebase) and delivered to the developer via a private Telegram notification. Feedback data is used solely to improve the app and is not shared with any third party.

Third-Party Services

The app communicates with the following external services:

  • mp3quran.net — Quran audio streaming. Please refer to their privacy policy for how they handle streaming requests.
  • Firebase Analytics (Google) — Anonymous usage analytics. Data is processed by Google under the Google Privacy Policy.
  • Cloud Firestore (Google) — Stores optional user feedback messages. No personal data is attached.
  • Telegram — Used by the developer to receive feedback notifications. No user data is stored on Telegram's servers beyond the notification content.
  • GitHub — App update check via a public endpoint. No personal data is sent.

Children's Privacy

The app does not collect any personal data from users of any age. The anonymous analytics data collected cannot be used to identify children or any individual user.

Changes to This Policy

If we update this privacy policy, the revised version will be published at the same URL with an updated date.
